FAQ

Frequently asked questions

Practical answers for setting up a fkst session, sending it work, finding the results, and controlling the session lifecycle from GitHub.

Session basics

What is fkst?
fkst is an autonomous coding platform. You describe work as GitHub issues, and an agent implements each approved item as a pull request in the same repository.
How do I start a session?
Open a GitHub issue labelled fkst-substrate-trigger. In the issue body, declare a Session Name, the Packages to run, and a Work Label that will identify the issues this session should claim.
How do I give a session work?
Open normal GitHub issues that carry that session's Work Label. fkst treats each labelled work issue as its own delivery item and works it through a separate pull request.

Results and logs

Where do the results appear?
Results appear as pull requests on the repository. Depending on the session and repository configuration, those pull requests may be reviewed manually or auto-merged, and they close the linked work issue when the delivery is complete.
How do I see a session's logs?
Every session streams redacted logs. The session's registration comment links to a secure, access-controlled download so authorized users can inspect the run without exposing secrets in public issue text.

Session lifecycle

How do I stop a session?
Close the trigger issue to stop the session permanently. If you only want the session to go idle, close or finish all open work issues carrying its Work Label.
