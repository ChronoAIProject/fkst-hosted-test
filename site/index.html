What is fkst

An engine that writes code from your backlog.

fkst pairs the fkst-substrate engine — an event-driven loop that runs packaged 'departments' of work — with a cloud control plane deployed on Kubernetes. Install the GitHub App on a repository and fkst treats that repo's issues as a work queue.

Each open issue is claimed by exactly one isolated agent. It reads the issue, plans and writes the change with an LLM coding agent, and opens a pull request for review. GitHub is the only durable state — issues are the queue, labels are the claims, and pull requests are the output.

issue → isolated pod → LLM writes code → pull request

Platform capabilities

Built for autonomous delivery.

  • Isolated per-session pods

    Every session runs in its own hard-isolated Kubernetes pod. One issue, one sandbox — no shared filesystem, no shared credentials, no blast radius between agents.

  • Issue → PR automation

    The control plane reconciles your open trigger issues, spawns an agent, and turns the work into a pull request. Issues are the queue, labels are the claim, PRs are the output.

  • Concurrent sandboxed agents

    Run many agents at once. Each claims a distinct issue via a label, works in parallel, and never touches another session's state.

  • Self-hosted control plane

    A Rust/Axum service on your own Kubernetes. It spawns and reaps pods, rotates each pod's GitHub token, and exposes a named-environments API. Your code and secrets stay in your cluster.

How it works

From issue to PR in three steps.

  1. 01

    Open a trigger issue

    Install the fkst GitHub App on your repo and open the trigger issue. The control plane starts watching that repository and reconciling its open issues.

  2. 02

    File your work issues

    Describe what you want as ordinary GitHub issues. Each one enters the queue and is claimed by a fresh, isolated agent in its own pod.

  3. 03

    Review the pull requests

    The agent writes the code with an LLM and opens a pull request. You review and merge — the same flow your team already uses.
